# Retail-Sales
This project focuses on analyzing retail sales data using SQL (PostgreSQL). The objective of this analysis is to perform data cleaning, data exploration, and business-driven analysis to extract meaningful insights from sales transactions.
The dataset contains information related to sales transactions such as customer demographics, product categories, sales amounts, and transaction timestamps.
Using SQL queries, the project answers key business questions related to sales performance, customer behavior, time-based trends, and category-wise analysis.
This project demonstrates strong command over:
SQL querying
Data cleaning techniques
Aggregations and grouping
Window functions
Common Table Expressions (CTEs)
Business problem solving using SQL

# Data Cleaning
Removed records with NULL values in critical columns
Ensured data consistency for analysis

# Data Exploration
Total number of sales
Unique customers count
Available product categories

# Key Insights
Identified best performing months per year
Analyzed customer purchase behavior by category and gender
Discovered peak sales shifts during the day
Highlighted top revenue-generating customers

# Conclusion
This project showcases practical SQL skills required for a Data Analyst role, including data cleaning, aggregation, window functions, and business-focused analysis.
It serves as a strong portfolio project for entry-level and junior data analyst positions.
